平台操作

Step 1 — 注册与登录

  1. 访问 https://sub2.pigcoder.com 打开控制台。
  2. 使用邮箱或第三方账号（GitHub / Google）完成注册。
  3. 登录后进入个人仪表盘。

Step 2 — 获取 API Key

  1. 进入控制台后，点击左侧菜单「令牌」。
  2. 点击「创建令牌」按钮。
  3. 为令牌填写一个名称（如 my-cli-key）。
  4. 根据需要选择令牌分组：
    • Anthropic 分组 — 用于 Claude Code 等 Anthropic 协议工具
    • OpenAI 分组 — 用于 Codex CLI、OpenCode 等 OpenAI 兼容工具
    • Gemini 分组 — 用于 Gemini CLI
  5. 点击「提交」创建令牌。
  6. 复制并妥善保存生成的 API Key。

注意：API Key 仅在创建时显示一次，请务必立即复制保存。如果丢失，需要重新创建新的令牌。

Step 3 — 确认端点地址

根据不同的 CLI 工具，使用对应的端点地址：

协议类型 端点地址 适用工具
Anthropic https://sub2.pigcoder.com Claude Code
OpenAI https://sub2.pigcoder.com/v1 Codex CLI, OpenCode, Droid CLI
Gemini https://sub2.pigcoder.com/v1beta Gemini CLI

Step 4 — 查看用量

登录控制台后，在首页仪表盘即可查看当前额度余额和调用统计。点击左侧菜单「日志」可以查看详细的请求记录与用量明细。

Step 5 — 使用密钥快速配置

获取 API Key 后，最快的方式是通过环境变量配置。在终端中执行：

bash 
# Anthropic 协议（Claude Code）
export ANTHROPIC_API_KEY="sk-pig-xxxx"
export ANTHROPIC_BASE_URL="https://sub2.pigcoder.com"

# OpenAI 协议（Codex CLI / OpenCode / Droid CLI）
export OPENAI_API_KEY="sk-pig-xxxx"
export OPENAI_BASE_URL="https://sub2.pigcoder.com/v1"

# Gemini 协议（Gemini CLI）
export GEMINI_API_KEY="sk-pig-xxxx"

提示：建议将环境变量写入 ~/.bashrc 或 ~/.zshrc，这样每次打开终端都会自动加载。

Step 6 — CC Switch 一键导入

如果你使用 CC Switch，可以通过 Deep Link 一键导入服务商配置，无需手动编辑任何文件：

text 
ccswitch://provider/add?name=Pigcoder&apiBaseUrl=https://sub2.pigcoder.com&apiKey=sk-pig-xxxx

在浏览器中打开上述链接，CC Switch 将自动添加 Pigcoder 作为服务商，填入你的 API Key 即可。

CLI 工具配置

Claude Code

Anthropic 官方推出的 AI 编程助手。运行依赖 Node.js (v18+)。

安装

bash 
# macOS / Linux
npm install -g @anthropic-ai/claude-code

# Windows (PowerShell 管理员)
npm install -g @anthropic-ai/claude-code

方式一：通过配置文件（推荐）

编辑 settings.json：

  • macOS / Linux：~/.claude/settings.json
  • Windows：%USERPROFILE%\.claude\settings.json
json 
{
  "apiKeyHelper": "echo sk-pig-xxxx",
  "apiBaseUrl": "https://sub2.pigcoder.com"
}

方式二：通过环境变量

bash 
# macOS / Linux
export ANTHROPIC_API_KEY="sk-pig-xxxx"
export ANTHROPIC_BASE_URL="https://sub2.pigcoder.com"
powershell 
# Windows (PowerShell)
$env:ANTHROPIC_API_KEY = "sk-pig-xxxx"
$env:ANTHROPIC_BASE_URL = "https://sub2.pigcoder.com"
cmd 
# Windows (CMD)
set ANTHROPIC_API_KEY=sk-pig-xxxx
set ANTHROPIC_BASE_URL=https://sub2.pigcoder.com

方式三：VS Code 扩展

如果你使用 VS Code 的 Claude Code 扩展，在 VS Code 设置中搜索 claude，然后修改以下配置：

  • API Key：填入 sk-pig-xxxx
  • API Base URL：填入 https://sub2.pigcoder.com

注意：Claude Code 使用 Anthropic 原生协议，端点地址不要加 /v1。

Codex CLI

OpenAI 官方推出的开源命令行编程助手。使用 OpenAI 兼容协议。

安装

bash 
npm i -g @openai/codex

配置文件

编辑 ~/.codex/config.toml：

toml 
[model]
name = "claude-sonnet-4-20250514"

[provider]
name = "openai"
base_url = "https://sub2.pigcoder.com/v1"

API Key 配置

编辑 ~/.codex/auth.json：

json 
{
  "openai": "sk-pig-xxxx"
}

Gemini CLI

Google 官方推出的 AI 命令行工具。使用 Gemini 协议。

安装

bash 
npm i -g @google/gemini-cli

方式一：通过环境变量

在 ~/.gemini/.env 中添加：

env 
GEMINI_API_KEY=sk-pig-xxxx
GEMINI_API_BASE_URL=https://sub2.pigcoder.com/v1beta

方式二：通过配置文件

编辑 ~/.gemini/settings.json：

json 
{
  "apiKey": "sk-pig-xxxx",
  "apiBaseUrl": "https://sub2.pigcoder.com/v1beta"
}

OpenCode

Go 语言编写的终端 AI 编程助手。使用 OpenAI 兼容协议。

安装

bash 
# macOS (Homebrew)
brew install opencode

# Go 安装
go install github.com/opencode-ai/opencode@latest

配置文件

在项目根目录创建 opencode.json：

json 
{
  "providers": {
    "pigcoder": {
      "apiKey": "sk-pig-xxxx",
      "baseURL": "https://sub2.pigcoder.com/v1",
      "models": {
        "claude-sonnet-4-20250514": {
          "maxTokens": 8096,
          "contextWindow": 200000
        }
      }
    }
  },
  "agents": {
    "coder": {
      "model": "pigcoder/claude-sonnet-4-20250514",
      "maxTokens": 8096
    },
    "task": {
      "model": "pigcoder/claude-sonnet-4-20250514",
      "maxTokens": 8096
    },
    "title": {
      "model": "pigcoder/claude-sonnet-4-20250514",
      "maxTokens": 200
    }
  }
}

轻量级 AI 终端编程助手，支持多种模型后端。使用 OpenAI 兼容协议。

安装

npm i -g droid-cli

配置文件

json 
{

  "apiKey": "sk-pig-xxxx",
  "baseURL": "https://sub2.pigcoder.com/v1"
}

CC Switch

Claude Code 服务商管理工具，可以在多个 API 服务商之间快速切换，无需手动修改配置文件。

核心功能

  • 一键切换 Claude Code 的 API 服务商
  • 支持管理多个服务商配置
  • 支持 Deep Link 一键导入服务商
  • 自动备份和恢复配置

安装

bash 
# macOS
brew install nicepkg/tap/cc-switch

# npm
npm i -g cc-switch

Deep Link 一键导入

在浏览器中打开以下链接即可自动导入 Pigcoder 配置：

text 
ccswitch://provider/add?name=Pigcoder&apiBaseUrl=https://sub2.pigcoder.com&apiKey=sk-pig-xxxx

手动配置

也可以通过命令行手动添加：

bash 
# 添加服务商
cc-switch provider add Pigcoder --api-base-url https://sub2.pigcoder.com --api-key sk-pig-xxxx

# 切换到 Pigcoder
cc-switch use Pigcoder

# 查看当前服务商
cc-switch current

故障排查

命令未找到（command not found）

安装后终端提示命令不存在，通常是因为 npm 全局 bin 目录不在系统 PATH 中。

解决方法：

# 查看 npm 全局 bin 目录
npm config get prefix

# 将输出的路径/bin 添加到 PATH
export PATH="$(npm config get prefix)/bin:$PATH"

API 连接失败

请求超时或无法连接到服务端。

解决方法：

  • 检查网络连接是否正常
  • 确认端点地址是否正确（参考上方端点地址表格）
  • 检查是否有代理或 VPN 阻断了连接
  • 尝试用 curl 测试端点连通性

端点配置错误

配置了错误的端点地址导致请求失败。

解决方法：

  • Claude Code 使用 Anthropic 协议，端点为 https://sub2.pigcoder.com，不要加 /v1
  • Codex CLI / OpenCode / Droid CLI 使用 OpenAI 协议，端点为 https://sub2.pigcoder.com/v1
  • Gemini CLI 使用 Gemini 协议，端点为 https://sub2.pigcoder.com/v1beta

API Key 无效

提示认证失败或 API Key 无效。

解决方法：

  • 确认 API Key 是否正确复制（没有多余空格或缺少字符）
  • 确认 API Key 的分组与所使用的工具匹配（Anthropic 分组对应 Claude Code，OpenAI 分组对应 Codex CLI 等）
  • 登录控制台查看令牌是否仍然有效，未被删除或过期
  • 检查账户余额是否充足
